Leady

Leady is an app I created that parses lead data and stores it in a MySQL DB that is later extracted for an automotive lead management app. The lead first comes in via email as XML and I parse the data to JSON, extract the required information and insert it into the DB. There's currently no front-end yet for this project as it wasn't necessary. This is my first real world solo project.


Challenges


  • One of the largest challenges with this project is related to the format in which leads are received. They do come in as XML and more specifically a format called "ADF" (Auto-lead Data Format). The problem is for whatever reason, lead providers such as Auto Trader deviate slightly from the format when sending data. This lead me having to utilize a lot of conditionals in orders to ensure data was parsed correctly.
  • Another issue is that new lead providers can be introduced at any point. This means there could potentially be another format I may need to consider. To address this, I implemented logging when an unknown provider is detected along with the request.
  • I have now been tasked with implementing a front-end for this project although it works well as just a backend. The expectation is that the user will be able to create templates as new formats come in. The challenge here is I have to have my code adapt to different variations of the XML ADF "standard" and have a process I was doing manually now be automated.

Technologies Used


  • JavaScript
  • JQuery
  • Node Modules - Imap, Xml-parser, Cron
  • Node.js
  • MySQL
